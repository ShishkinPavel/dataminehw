# Финальный отчёт: Sentiment Analysis Pipeline

## 1. Описание задачи и датасета

**Задача:** бинарная классификация тональности (positive/negative) отзывов на фильмы.

**Датасет:** 5998 отзывов из двух источников:
- **IMDB** (HuggingFace datasets library) — 5000 строк, длинные рецензии
- **Rotten Tomatoes** (HuggingFace REST API) — 1000 строк, короткие рецензии

**Распределение GT-меток:** 3505 positive (58.4%) / 2493 negative (41.6%) — умеренно сбалансированный датасет (ratio=0.71).

## 2. Что делал каждый агент

### DataCollectionAgent (HW1)
Собрал данные из двух источников, соответствующих требованию HW1:
1. HuggingFace `datasets` library — загрузка IMDB (5000 строк) через `_load_dataset()`
2. HuggingFace Datasets REST API — загрузка Rotten Tomatoes (1000 строк) через `_fetch_hf_api()`

Унифицировал схему: `text, label, source, collected_at`. Провёл EDA: распределение классов, длины текстов (медиана 840 символов), топ-20 слов, распределение по источникам.

### DataQualityAgent (HW2)
Обнаружил 2 дубликата (0.03%), остальные метрики чистые (0 пропусков, 0 выбросов). Вызвал **YandexGPT** для LLM-рекомендации стратегии чистки. Рекомендация совпала с выбором пользователя: удалить дубликаты. Стратегия через HITL: `drop/drop/clip_iqr`.

### AnnotationAgent (HW3)
Запустил BART `facebook/bart-large-mnli` на стратифицированной выборке 200 строк для аудита GT-меток. Результаты:
- Cohen's kappa = 0.711 (хорошее согласие)
- 29 disagreements (14.5%), из них 24 — BART переразметил positive → negative
- BART систематически ошибается на отзывах с негативной лексикой в позитивном контексте

Сгенерировал спецификацию разметки и экспорт в LabelStudio.

### ActiveLearningAgent (HW4)
Сравнил 3 стратегии: entropy, least_confidence, random. Entropy/LC идентичны при 2 классах и значительно превосходят random (F1=0.7630 vs 0.6998 на 250 примерах). Вызвал **YandexGPT** для анализа стратегий и объяснения выбора примеров.

## 3. Описание HITL-точки

### Основная HITL-точка: Annotation Review (Шаг 3)

**Контекст:** BART zero-shot классификатор показал 29 disagreements с GT на выборке 200 строк. Пользователю показаны 4 примера с наивысшим confidence BART (>0.95).

**Проверено:** 4 примера (все — high-confidence ошибки BART)

**Результат:**
| # | GT | BART | Conf | Решение пользователя |
|---|-----|------|------|---------------------|
| 1 | positive | negative | 0.982 | **positive** — согласие с GT |
| 2 | negative | positive | 0.981 | **negative** — согласие с GT |
| 3 | positive | negative | 0.965 | **positive** — согласие с GT |
| 4 | positive | negative | 0.955 | **positive** — согласие с GT |

**Исправлено:** 0 меток. Все GT метки подтверждены пользователем. BART ошибался на всех 4 примерах.

**Вывод:** GT-метки IMDB высокого качества. BART полезен как инструмент аудита для нахождения edge cases, но ненадёжен как автоматический разметчик при наличии негативной лексики в позитивном контексте.

### Дополнительные HITL-точки
- **Шаг 1:** выбор размера выборки (5000 IMDB + 1000 RT)
- **Шаг 2:** выбор стратегии чистки (drop/drop/clip_iqr) + рекомендация YandexGPT
- **Шаг 3:** выбор типа меток для обучения (GT, рекомендовано)
- **Шаг 4:** подтверждение стратегий AL и сохранение модели

## 5. Ретроспектива

### Что получилось хорошо
- **Пайплайн работает end-to-end** — от сбора данных до обученной модели за один прогон
- **Active Learning эффективен** — entropy достигает F1≥0.70 на 70 примерах, random — только на 210 (экономия 67%)
- **HITL-точки осмысленны** — annotation review подтвердил высокое качество GT-меток
- **YandexGPT интеграция** — LLM-рекомендации на шагах 2 и 4 дают полезный контекст
- **Хорошие метрики** — F1=0.857 на baseline (TF-IDF + LogReg) — достойный результат

### Что можно улучшить
- **BART на полном датасете** — запуск на выборке 200 строк экономит время, но даёт неполную картину ошибок
- **Нейтральный класс** — бинарная классификация теряет нюансы смешанных отзывов
- **Feature engineering** — TF-IDF baseline хорош, но transformer-based модель (BERT) даст F1 > 0.90
- **Cross-domain** — модель обучена только на кинорецензиях, нужна валидация на других доменах
- **Дисбаланс** — 58/42 не критичен, но при более сильном дисбалансе нужны oversampling/undersampling

### Уроки
1. GT-метки из IMDB надёжнее BART zero-shot (kappa=0.71). Для текстовой классификации человеческие метки остаются золотым стандартом
2. Entropy и least_confidence идентичны при 2 классах — нет смысла сравнивать оба
3. BART систематически ошибается на отзывах с негативной лексикой в позитивном контексте — это известная проблема zero-shot моделей
4. Больше данных = лучше модель: F1 выросла с ~0.76 (250 примеров AL) до 0.857 (4798 примеров full train)
